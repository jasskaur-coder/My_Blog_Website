Git & GitHub

Let’s talk about two tools that changed the way developers collaborate: Git and GitHub.

Why Version Control?

  • Track changes in your code
  • Revert if something breaks
  • Collaborate safely
  • Keep your code online & backed up

What is Git?

Git is a local version control tool that helps track changes in your code.

  • Installed on your system
  • Lets you commit, rollback, and branch

What is GitHub?

GitHub is a cloud platform to host your Git repositories.

  • Share your code (public or private)
  • Work with teams or contribute to projects
  • Store documentation like README, Issues, etc.

Think of Git as your tool, and GitHub as where your project lives online.

Git Workflow

        Write Code → Save File → git add → git commit → git push → GitHub

Basic Git Concepts

  • Repository: Folder tracked by Git
  • Commit: Save point of your code
  • Branch: A separate version
  • Merge: Combine branches
  • Clone: Download repo from GitHub
  • Push: Upload changes to GitHub
  • Pull: Get latest from GitHub

Real-Life Example

Building a portfolio site?

  • You start a Git repo
  • Every change = one commit
  • Push to GitHub to share with others

Quick Tip

Always add a README.md with:

  • What your project does
  • How to run it
  • Features or screenshots
